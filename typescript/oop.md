# Imperative and Procedurlal Programming

: 절차 지향적 프로그래밍

- 어플리케이션이 동작하기 위한 데이터와 함수들 위주로 구성하는 프로그래밍 방식
- 가독성이 떨어지고 유지보수, 확장이 어렵다.

# Object Oriented Programming

: 객체지향 프로그래밍

- Object단위로 만들어간다. 유지보수 쉽다. 재사용성이 높다. 확장성 높다.

## Object 구성

- 데이터(상태) = properties (또는 fields)
- 함수(상태 변화시킴) = methods

## 객체지향 원칙 4가지

- Encapsulation
- Abstraction
- Inheritance
- Polymorphism
